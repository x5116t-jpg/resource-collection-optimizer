# ResouceCollection_05 プロジェクト全体配布パッケージ

## 📦 配布パッケージ構成

プロジェクト全体を配布する場合の推奨構成：

```
ResouceCollection_05_Distribution/
│
├── 📄 README.md (プロジェクト説明)
├── 📄 SETUP_GUIDE.md (セットアップガイド)
├── 🪟 setup.bat (Windows用セットアップ)
├── 🍎 setup.sh (Mac/Linux用セットアップ)
│
├── project/ (プロジェクト本体)
│   ├── src/ (ソースコード)
│   ├── scripts/ (スクリプト)
│   ├── tests/ (テスト)
│   ├── data/ (データファイル)
│   ├── docs/ (ドキュメント)
│   ├── claudedocs/ (Claude生成ドキュメント)
│   ├── networkx/ (依存モジュール)
│   ├── *.bat (起動スクリプト)
│   └── requirements.txt (依存関係リスト)
│
└── superclaude/ (SuperClaude設定)
    ├── installer.bat (設定インストーラー)
    ├── installer.command (Mac用)
    ├── config_files/ (.mdファイル)
    └── docs/ (使い方ガイド)
```

## 🚀 セットアップスクリプト

### setup.bat (Windows用 統合セットアップ)

```batch
@echo off
chcp 65001 > nul
echo.
echo ====================================
echo ResouceCollection_05
echo 統合セットアップ
echo ====================================
echo.
echo このスクリプトは以下を自動的に行います:
echo 1. Python環境の確認
echo 2. 仮想環境の作成
echo 3. 依存関係のインストール
echo 4. SuperClaude設定の適用
echo.
pause

REM ステップ1: Python確認
echo.
echo [1/4] Python環境を確認中...
python --version >nul 2>&1
if %errorlevel% neq 0 (
    echo ❌ エラー: Pythonがインストールされていません。
    echo.
    echo Python 3.8以上をインストールしてください:
    echo https://www.python.org/downloads/
    pause
    exit /b 1
)
echo ✅ Python が見つかりました
python --version

REM ステップ2: 仮想環境作成
echo.
echo [2/4] 仮想環境を作成中...
cd project
if exist ".venv" (
    echo ⚠️  既存の仮想環境が見つかりました
    choice /C YN /M "削除して再作成しますか"
    if errorlevel 2 goto skip_venv
    rmdir /s /q .venv
)
python -m venv .venv
if %errorlevel% neq 0 (
    echo ❌ 仮想環境の作成に失敗しました
    pause
    exit /b 1
)
echo ✅ 仮想環境作成完了

:skip_venv

REM ステップ3: 依存関係インストール
echo.
echo [3/4] 依存関係をインストール中...
call .venv\Scripts\activate.bat
if exist "requirements.txt" (
    pip install -r requirements.txt
    echo ✅ 依存関係インストール完了
) else (
    echo ⚠️  requirements.txt が見つかりません
    echo 依存関係のインストールをスキップします
)

REM ステップ4: SuperClaude設定
echo.
echo [4/4] SuperClaude設定を適用中...
cd ..\superclaude
call installer.bat
cd ..\project

echo.
echo ====================================
echo ✅✅✅ セットアップ完了！ ✅✅✅
echo ====================================
echo.
echo 次のステップ:
echo 1. プロジェクトフォルダ内で作業を開始
echo 2. Claude Code を起動
echo 3. この project フォルダを開く
echo.
echo 起動スクリプト:
echo - ClaudeCode_start.bat  (Claude Code起動)
echo - run_app.bat           (アプリケーション実行)
echo.
pause
```

### setup.sh (Mac/Linux用)

```bash
#!/bin/bash

echo ""
echo "===================================="
echo "ResouceCollection_05"
echo "統合セットアップ"
echo "===================================="
echo ""
echo "このスクリプトは以下を自動的に行います:"
echo "1. Python環境の確認"
echo "2. 仮想環境の作成"
echo "3. 依存関係のインストール"
echo "4. SuperClaude設定の適用"
echo ""
read -p "続行するには Enter キーを押してください..."

# ステップ1: Python確認
echo ""
echo "[1/4] Python環境を確認中..."
if ! command -v python3 &> /dev/null; then
    echo "❌ エラー: Python3がインストールされていません。"
    echo ""
    echo "Python 3.8以上をインストールしてください:"
    echo "https://www.python.org/downloads/"
    exit 1
fi
echo "✅ Python が見つかりました"
python3 --version

# ステップ2: 仮想環境作成
echo ""
echo "[2/4] 仮想環境を作成中..."
cd project
if [ -d ".venv" ]; then
    echo "⚠️  既存の仮想環境が見つかりました"
    read -p "削除して再作成しますか? (y/n): " -n 1 -r
    echo
    if [[ $REPLY =~ ^[Yy]$ ]]; then
        rm -rf .venv
    else
        echo "既存の仮想環境を使用します"
    fi
fi

if [ ! -d ".venv" ]; then
    python3 -m venv .venv
    if [ $? -ne 0 ]; then
        echo "❌ 仮想環境の作成に失敗しました"
        exit 1
    fi
    echo "✅ 仮想環境作成完了"
fi

# ステップ3: 依存関係インストール
echo ""
echo "[3/4] 依存関係をインストール中..."
source .venv/bin/activate
if [ -f "requirements.txt" ]; then
    pip install -r requirements.txt
    echo "✅ 依存関係インストール完了"
else
    echo "⚠️  requirements.txt が見つかりません"
    echo "依存関係のインストールをスキップします"
fi

# ステップ4: SuperClaude設定
echo ""
echo "[4/4] SuperClaude設定を適用中..."
cd ../superclaude
chmod +x installer.command
./installer.command
cd ../project

echo ""
echo "===================================="
echo "✅✅✅ セットアップ完了！ ✅✅✅"
echo "===================================="
echo ""
echo "次のステップ:"
echo "1. プロジェクトフォルダ内で作業を開始"
echo "2. Claude Code を起動"
echo "3. この project フォルダを開く"
echo ""
read -p "終了するには Enter キーを押してください..."
```

## 🛠️ 配布パッケージ作成手順

### 手順1: requirements.txt の作成

```bash
cd "D:\py\Resource Collection\ResouceCollection_05"
pip freeze > requirements.txt
```

または、必要なパッケージのみリストアップ:

```text
# requirements.txt
networkx>=3.0
# その他必要なパッケージ
```

### 手順2: 配布フォルダの作成

```powershell
# PowerShellで実行

# ベースフォルダ作成
New-Item -ItemType Directory -Path "ResouceCollection_05_Distribution"
New-Item -ItemType Directory -Path "ResouceCollection_05_Distribution\project"
New-Item -ItemType Directory -Path "ResouceCollection_05_Distribution\superclaude"

# プロジェクトファイルをコピー（不要なファイル除外）
$exclude = @('.venv', '.pytest_cache', '__pycache__', '*.pyc', 'SuperClaude_Setup')

Get-ChildItem "D:\py\Resource Collection\ResouceCollection_05" -Exclude $exclude |
    Copy-Item -Destination "ResouceCollection_05_Distribution\project" -Recurse -Force

# SuperClaude設定をコピー
Copy-Item "D:\py\Resource Collection\ResouceCollection_05\SuperClaude_Setup\*" `
    -Destination "ResouceCollection_05_Distribution\superclaude" -Recurse -Force
```

### 手順3: セットアップスクリプトの配置

上記の `setup.bat` と `setup.sh` を作成して配置

### 手順4: ドキュメントの作成

- README.md
- SETUP_GUIDE.md
- LICENSE (必要に応じて)

### 手順5: テスト

1. 別のフォルダに配布パッケージをコピー
2. setup.bat / setup.sh を実行
3. 正しく動作するか確認

### 手順6: ZIP化

```powershell
Compress-Archive -Path "ResouceCollection_05_Distribution" `
    -DestinationPath "ResouceCollection_05_v1.0.zip"
```

## 🎯 推奨配布方法

1. **ZIP圧縮** - 最もシンプル
2. **Git リポジトリ** - バージョン管理が必要な場合
3. **Docker イメージ** - 環境の完全な再現が必要な場合

ターミナル未経験者には **ZIP + 自動セットアップスクリプト** が最適です。
